# Agents

This file provides guidance to AI assistants (aka agents) when working with code in this repository.

## Project Overview

Terraform module for managing Google Cloud Monitoring resources. Each resource type lives in its own submodule under `modules/` with a corresponding example under `examples/`.

## Development Setup

```shell
mise install  # Install tools: terraform, terraform-docs, tflint, gcloud, prek
prek install  # Install pre-commit hooks (terraform_docs, terraform_fmt, terraform_tflint)
tflint --init # Initialize tflint plugins
gcloud auth application-default login --project=terraform-test
```

Provider credentials go in `.env` (autoloaded by mise): `GOOGLE_PROJECT` and `GOOGLE_REGION`.

## Commands

Test a specific module (each module has its own `basic.tftest.hcl`):
```shell
cd modules/<module-name>
terraform init && terraform test
```

Validate:
```shell
terraform init && terraform validate
```

Format check:
```shell
terraform fmt -check
```

Lint:
```shell
tflint
```

Regenerate module READMEs (autoruns via pre-commit hook):
```shell
terraform-docs markdown table modules/<module-name>
```

## Architecture

- `modules/<name>/` — self-contained submodules, each wrapping a single `google_monitoring_*` resource with `for_each` over a map variable
- `examples/<name>/` — runnable examples referencing `../../modules/<name>` (used for manual convergence testing)
- `modules/<name>/basic.tftest.hcl` — Terraform native tests (run blocks with assertions, auto-teardown)
- Module READMEs are auto-generated by `terraform-docs` between `<!-- BEGIN_TF_DOCS -->` markers

## Conventions

- Conventional commits: `feat(module-name):`, `fix(module-name):`, `docs(readme):`, `chore(main):`
- Releases managed by release-please (`.github/workflows/release.yml`)
- Resource instances keyed as `google_monitoring_<type>.default` with `for_each`
- Output pattern: `<type>_ids` map keyed by the input map keys
